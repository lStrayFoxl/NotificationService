# Notification-service
Сервис предоставляет API для отправки уведомлений и получения уведомлений пользователя.

## Инструкция настройке и запуску сервиса
- Склонировать репозиторий на локальную машину;
- Выполнить команду 'make setup_dev' для подготовки сервиса к запуску;
- Запустить сервис командой 'make run_dev';
- Создать БД для проекта:
  * Перейти в PGAdmin(http://localhost:8080/);
  * Добавить сервер PostgreSQL, через кнопку 'Add New Server':
    1. Во вкладке 'General', необходимо указать любое имя сервер(поле Name);
    2. Во вкладке 'Connection', необходимо указать хост(поле Host name/adress, значение можно взять в env файле - DB_HOST), имя пользователя(поле Username, значение можно взять в env файле - DB_USERNAME), пароль(поле Password, значение можно взять в env файле - DB_PASSWORD);
    3. Нажать кнопку 'Save';
  * В добавленном сервере, во вкладке 'Databases', создаём новую БД:
    1. Во вкладке 'General', необходимо указать имя БД(поле Database, значение можно взять из env - DB_DATABASE или указать своё, не забыв обновить env);
    2. Нажать кнопку 'Save';
- Выполнить команду 'make init_db_and_rabbitmq', для подготовки БД и RabbitMQ к работе;
- Сервис готов к работе;
  * В дальнейшем для запуска сервиса достаточно использовать только команду 'make run_dev';

## Инструкция по выключению сервиса
- Для выключения сервиса можно использовать команду 'make stop';

## Тестирование работы сервиса
Для тестирования сервиса можно использовать Postman. Конфигурация коллекции находится в корне проекта(имя файла конфигурации: Notification Service.postman_collection.json).

Коллекция содержит 2 тестовых сценария:
- Отправка уведомления указанным пользователям по указанному каналу(sms, email) - Send Notifications;
- Получение списка уведомлений указного пользователя - Get User Notifications;

## Используемый стек технологий
- PHP(Laravel);
- PostgreSQL;
- RabbitMQ;
- Redis;
- Docker;
